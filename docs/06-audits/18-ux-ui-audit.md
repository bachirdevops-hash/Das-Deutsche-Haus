# 🎪 UX / UI Audit

## Strengths
- ✅ Strong DDH brand identity (red/gold/black flag colors consistent)
- ✅ Cinematic hero slideshow creates emotional connection
- ✅ RTL-aware throughout
- ✅ Modal flows for lead capture are simple (name/email/phone)
- ✅ Clear admin role indicators
- ✅ Polished cards with hover lift

## Heuristic Issues (Nielsen 10)
| # | Heuristic | Issue |
|---|-----------|-------|
| 1 | Visibility of system status | Some long-running actions (build) lack progress |
| 2 | Match real world | "Convert to user" jargon — maybe "إنشاء حساب للطالب" reads better |
| 3 | Consistency | 3 different button styles in same area sometimes |
| 4 | Error prevention | No "are you sure?" before deleting leads |
| 5 | Recognition | Some admin tabs have only emoji, no consistent icons |
| 6 | Flexibility | No keyboard shortcuts for power users |
| 7 | Aesthetic | Featured Event card layout could overflow on tiny screens |
| 8 | Help recovery | Form error states use toast only — inline errors better |
| 9 | Help & docs | No tooltips on admin field meanings (e.g. icon names) |
| 10 | Microcopy | Inconsistent voice (sometimes formal Arabic, sometimes casual) |

## Conversion Recommendations
1. **Sticky CTA bar on mobile** for course pages (always show "سجّل الآن")
2. **Social proof above CTA**: "4,200+ تخرّجوا معنا"
3. **Trust badges** near pricing (telc logo, Goethe logo)
4. **Live chat or WhatsApp pulse** (already exists, could be more prominent)
5. **Exit-intent modal** with discount offer
6. **Testimonials with video** (higher trust than text)
7. **A/B test hero CTAs** — "Apply Now" vs "Discover Courses"
